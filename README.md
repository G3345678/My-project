# 校园信息管理系统

基于 Rust + egui 开发的桌面端校园信息管理工具，支持学生/教师/图书三大模块，具备大规模数据处理能力。

## 技术栈

| 组件 | 技术 |
|------|------|
| 语言 | Rust 1.94 |
| GUI | eframe / egui 0.29（即时模式） |
| 序列化 | Serde + serde_json |
| CSV | csv 1.3 |
| 文件对话框 | rfd 0.12 |

## 核心功能

### 学生管理
- 基本信息 + 多科成绩增删改查
- 按学号/姓名/院系/专业模糊搜索
- 三种排序：录入顺序 / 均分降序 / 班级升序
- 单个学生成绩单 CSV 导出（含 A/B/C/D/F 等级评定）
- 批量选择删除

### 教师管理
- 工号、姓名、院系、职称、研究方向管理
- 模糊搜索与分页浏览

### 图书管理
- ISBN、书名、作者、出版社、馆藏数量管理
- **低库存预警系统**（可配置阈值，6 处 UI 联动）
- 预警列：缺货/低库存/正常三色标记
- 低库存报告 CSV 一键导出

### 数据统计
- 学生男女比例（含百分比）、平均年龄、院系分布
- **院系 × 性别交叉统计表**
- 教师职称分布
- 书籍分类分布 + 低库存预警汇总

### 数据持久化
- JSON 全量快照 + AOF 增量日志双保险
- 启动自动恢复，异常退出不丢数据
- JSON / CSV 多格式导入导出

## v2.0.0 性能优化

| 优化项 | 原版 | 优化版 | 效果 |
|--------|------|--------|------|
| ID 查重 | `iter().any()` O(n) | HashMap `contains_key()` | **O(n)→O(1)** |
| UI 渲染 | 全量渲染 N 行 | 分页固定 50 行/页 | 10 万数据不卡 |
| 过滤数据 | 每帧克隆 `Vec<Student>` | 缓存 `Vec<usize>` 索引 | 消除大对象克隆 |
| 过滤触发 | 每帧 60 次/秒 | 仅搜索词变化时 | 触发频率 ↓99% |
| 索引构建 | 无 | `reserve()` 预分配 | 消除 rehash 抖动 |

**数据承载**: 流畅支持 100,000 条记录，内存 ~140MB

## 运行

```bash
cargo run --release
```

首次运行自动创建 `campus_db.json`（快照）和 `campus_db.aof`（日志）。

## 项目结构

```
src/
├── main.rs      # 入口
├── lib.rs       # 数据模型、验证、持久化、索引
└── ui.rs        # GUI 布局与交互逻辑
```

## 导入数据格式

**学生 CSV**：`学号,姓名,性别,年龄,院系,专业,班级,课程,分数`（同一学号多行自动合并成绩）

**教师 CSV**：`工号,姓名,性别,年龄,院系,职称,研究方向`

**图书 CSV**：`ISBN,书名,作者,出版社,年份,分类,数量`
